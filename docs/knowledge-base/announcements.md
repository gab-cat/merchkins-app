# Announcements Management Guide

## Overview

The announcements system allows you to communicate important information to customers through banners, notifications, and in-app messages. Learn how to create, schedule, and track announcements.

## Creating Announcements

### Step 1: Navigate to Announcements

1. Go to Admin â†’ Announcements
2. Click "Create New Announcement"

### Step 2: Basic Information

**Required Fields:**

- **Title** - Announcement headline
- **Content** - Announcement message/content
- **Type** - Banner, Notification, or In-app

**Optional Fields:**

- **Description** - Additional details
- **Link** - URL to redirect (optional)
- **Image** - Visual content (optional)
- **Tags** - Organization tags

### Step 3: Targeting & Scheduling

**Targeting Options:**

- **All Customers** - Show to everyone
- **Specific Segments** - Target customer groups
- **Organization-Specific** - Show to your store customers

**Scheduling:**

- **Immediate** - Publish now
- **Scheduled** - Set start and end dates
- **Recurring** - Repeat announcement (if supported)

### Step 4: Configuration

**Display Settings:**

- **Priority** - Display priority (HIGH, MEDIUM, LOW)
- **Pinned** - Pin to top of announcements
- **Dismissible** - Allow customers to dismiss
- **Position** - Where to display (top, bottom, etc.)

**Visibility:**

- **Active** - Currently visible
- **Inactive** - Hidden from customers
- **Draft** - Not yet published

### Step 5: Save Announcement

1. Review all settings
2. Click "Create Announcement"
3. Announcement is published (if immediate)

## Scheduling Announcements

### Immediate Publishing

**Publish Now:**

- Announcement goes live immediately
- Visible to customers right away
- No scheduling needed

**Use Cases:**

- Urgent announcements
- Breaking news
- Immediate updates

### Scheduled Publishing

**Start Date:**

- Set when announcement starts
- Automatically publishes at start time
- Can schedule in advance

**End Date:**

- Set when announcement ends
- Automatically hides at end time
- Useful for time-limited announcements

**Use Cases:**

- Promotional campaigns
- Seasonal announcements
- Scheduled maintenance notices

## Pinning Announcements

### Pinned Announcements

**Pin to Top:**

- Pinned announcements appear first
- Always visible (unless dismissed)
- Use for important messages

**Unpinning:**

- Remove pin to unpin
- Announcement appears in normal order
- Still active if within schedule

**Best Practices:**

- Pin only critical announcements
- Limit number of pinned items
- Unpin when no longer critical
- Use sparingly for maximum impact

## Frequently Asked Questions

### Q: How many announcements can be active at once?

**A:** There's no hard limit, but avoid showing too many simultaneously to prevent overwhelming customers.

### Q: Can I schedule announcements in advance?

**A:** Yes, set start and end dates when creating announcements. They'll publish automatically.

### Q: What's the difference between pinned and regular announcements?

**A:** Pinned announcements appear first and stay visible. Regular announcements appear in chronological order.

### Q: Can customers dismiss announcements?

**A:** Yes, if announcements are set as dismissible, customers can dismiss them. They won't see dismissed announcements again.

### Q: How do I track announcement performance?

**A:** Use the Analytics section to view views, clicks, engagement rates, and other metrics for each announcement.

### Q: Can I target specific customer segments?

**A:** Yes, you can target announcements to specific customer segments based on various criteria.
